KEY RESEARCH FINDINGS
=====================

FINDING 1: Dual Persistence System
- Checkpointers: Thread-scoped state snapshots (high-frequency updates)
- Stores: Cross-thread key-value data (lower-frequency updates)
- Both configurable at graph compilation time
- Can use different backends for each

FINDING 2: Multi-Turn Support
- Thread-based conversation context via unique thread_id
- MessagesState for conversation-style interactions
- Automatic state restoration between turns
- Message accumulation with optional summarization
- 7 streaming modes for real-time interaction

FINDING 3: State Recovery
- Checkpoints preserve state even on errors
- Interrupts enable human-in-the-loop workflows
- Time-travel debugging via checkpoint history
- Critical rules for safe interrupt patterns
- Multi-interrupt handling with resume maps

FINDING 4: Thread Management
- Hierarchical thread IDs: org:user:conversation
- Independent conversation contexts
- Agent Server abstracts infrastructure
- Unlimited multi-user support
- State access via get_state() and list_state()

FINDING 5: Production Deployment
- PostgreSQL recommended for durability
- Separate checkpointer/store backends
- Checkpoint latency: 10-100ms (database dependent)
- Async checkpointers for non-blocking I/O
- Comprehensive monitoring required

BEST PRACTICES DOCUMENTED
=========================

Safety
1. Never wrap interrupt() in try/except
2. Ensure pre-interrupt operations are idempotent
3. Place side effects after interrupt resumption
4. Use edges for loops, not interrupt loops in nodes
5. Maintain deterministic interrupt ordering

Performance
1. Use async checkpointers for non-blocking I/O
2. Stream long-running operations for responsiveness
3. Implement periodic checkpoint cleanup
4. Monitor checkpoint latency (target <100ms)
5. Use connection pooling for databases

Production
1. Use PostgreSQL for production (not InMemory)
2. Deploy separate checkpointer and store backends
3. Implement hierarchical thread IDs
4. Set up comprehensive monitoring
5. Test fault recovery procedures
6. Plan state size management strategy
7. Establish backup and replication procedures
8. Use Agent Server for managed infrastructure
